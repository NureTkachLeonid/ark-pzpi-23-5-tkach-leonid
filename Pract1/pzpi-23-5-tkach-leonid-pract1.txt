1. МЕТА
Імплементація стандартів Code Conventions (Google Style) для забезпечення надійності, читабельності та безпеки програмного коду IoT-системи.

2. ХІД РОБОТИ

2.1. Актуальність Code Style для IoT
У розробці систем Інтернету речей (IoT) якість коду має критичне значення. На відміну від звичайних програм, помилки в коді "Smart Plant Care" можуть призвести до фізичних наслідків: загибелі рослин через відсутність поливу або затоплення через несправність насоса. Використання єдиного стилю дозволяє мінімізувати такі ризики шляхом покращення розуміння коду всіма учасниками команди.

2.2. Застосовані правила іменування (Naming)
Було впроваджено чіткі правила для ідентифікаторів:
- **Змінні (camelCase):** Назви мають чітко описувати фізичний параметр.
  Приклад: `soilMoisture` (вологість ґрунту), `lightIntensity` (інтенсивність світла).
- **Класи (PascalCase):** Назви класів описують сутність реального світу або контролер.
  Приклад: `WaterPumpController` (контролер водяного насоса).

2.3. Використання констант
Для уникнення "магічних чисел", які роблять код незрозумілим, всі налаштування винесено в іменовані константи у форматі `UPPER_CASE`.
Приклад: Замість числа 100 у коді використовується константа `MAX_WATER_LEVEL`. Це дозволяє змінювати параметри в одному місці без ризику пропустити якесь значення.

2.4. Документація
Критично важливі методи, що відповідають за прийняття рішень (наприклад, `checkWateringNeed()`), задокументовані за допомогою формату Javadoc (`/** ... */`). Це дозволяє автоматично генерувати документацію та бачити підказки в IDE.

3. ПРИКЛАДИ КОДУ

3.1. Приклад 1: "Брудний код" (Before)
Фрагмент коду, де логіка прихована за незрозумілими назвами змінних.

void func(int a, int b) {
  if (a < 500) {
    if (b == 1) { pumpOn(); }
  }
}

3.2. Приклад 2: "Чистий код" (After)
Той самий фрагмент після застосування стандартів оформлення. Код став самодокументованим.

/**
 * Керує поливом на основі вологості.
 */
void manageWatering(int soilMoisture, int pumpState) {
  if (checkWateringNeed(soilMoisture)) {
    activatePump();
  }
}

ВИСНОВКИ
Стандартизація коду в проєкті "Smart Plant Care" значно спростила процес налагодження фізичних пристроїв. Впровадження правил іменування та документування підготувало проєкт до подальшого масштабування та командної розробки, знизивши вірогідність критичних помилок.
