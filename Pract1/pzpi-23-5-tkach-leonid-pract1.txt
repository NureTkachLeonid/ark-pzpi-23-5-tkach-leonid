1. МЕТА
Дослідити та застосувати правила стандартів кодування (Coding Conventions) для забезпечення якості програмного коду в системі Smart Plant Care.

2. ХІД РОБОТИ

2.1. Вибір стандарту кодування
Для розробки програмного забезпечення мікроконтролерів (C++) у проєкті "Smart Plant Care" було обрано стандарт "Google C++ Style Guide". Дотримання цього стандарту дозволяє зробити код більш читабельним, безпечним та сумісним з іншими бібліотеками. Єдиний стиль форматування мінімізує конфлікти при злитті гілок у системі контролю версій.

2.2. Правила іменування
- **Змінні:** Використовується стиль camelCase, де перше слово пишеться з маленької літери, а наступні - з великої. Назви мають бути англомовними та змістовними.
  Приклад: `soilMoisture` (вологість ґрунту), `lightIntensity` (інтенсивність світла).
- **Константи:** Використовується стиль UPPER_CASE з розділенням слів підкресленням.
  Приклад: `MAX_WATER_LEVEL` (максимальний рівень води), `SENSOR_READ_TIMEOUT`.
- **Класи:** Використовується стиль PascalCase.
  Приклад: `PlantController` (контролер рослини), `SensorModules`.

2.3. Форматування
- **Відступи:** Використовується 4 пробіли для позначення вкладеності блоків коду. Табуляція не використовується для уникнення проблем з відображенням у різних редакторах.
- **Дужки:** Відкриваюча фігурна дужка `{` ставиться на тому ж рядку, що й оголошення функції або циклу (стиль K&R / Google).

2.4. Коментарі
- Весь публічний інтерфейс класів (меди, що можуть викликатися ззовні) має бути задокументований.
- Використовується формат документації Javadoc/Doxygen: `/** ... */`.
- Коментарі пояснюють "що" робить метод і "чому", а не "як" (реалізація має бути зрозумілою з коду).

3. ПРИКЛАДИ КОДУ

3.1. Приклад "Поганого коду"
Даний фрагмент демонструє відсутність форматування, незрозумілі імена змінних та "магічні числа".

int a=5; void f(){if(a>0)x=1;} // Незрозумілі змінні, відсутність відступів

3.2. Приклад "Хорошого коду" (refactored)
Той самий логічний блок, але переписаний згідно зі стандартами.

/**
 * Перевіряє рівень води в резервуарі та вмикає насос за потреби.
 */
void checkWaterLevel() {
    if (currentWaterLevel < MIN_THRESHOLD) {
        logInfo("Water level low. Activating pump.");
        activatePump();
    }
}

ВИСНОВКИ
Використання єдиного стилю кодування спрощує командну розробку та знижує поріг входження нових розробників у проєкт Smart Plant Care. Чіткі правила іменування та форматування роблять код самодокументованим, що зменшує кількість помилок та полегшує процес Code Review.
