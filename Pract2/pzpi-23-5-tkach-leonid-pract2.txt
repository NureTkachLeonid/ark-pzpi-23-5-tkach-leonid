1. МЕТА
Навчитися застосовувати операції рефакторингу для покращення якості коду та отримати практичні навички роботи з гілками в системі контролю версій Git на прикладі проєкту "Smart Plant Care System".

2. ХІД РОБОТИ

2.1. Організація роботи з Git (Git Flow)
Для ефективної командної роботи та управління версіями проєкту використовується стратегія розгалуження Git Flow, яка включає наступні типи гілок:
- `main`: Гілка для релізів. Містить стабільний код, готовий до використання кінцевим користувачем. Зміни сюди потрапляють тільки після ретельного тестування.
- `develop`: Основна гілка розробки. Всі нові функції зливаються сюди. Це остання версія проєкту, яка готується до наступного релізу.
- `feature/Name`: Гілки для розробки нових функцій (наприклад, `feature/sensors`, `feature/auth`). Вони створюються від `develop` і зливаються назад після завершення роботи та Code Review.

2.2. Техніки рефакторингу
Рефакторинг — це процес зміни внутрішньої структури програми без зміни її зовнішньої поведінки. У цій роботі використано такі техніки:
- Extract Method (Виділення методу): Використовується, коли фрагмент коду можна згрупувати разом. Цей фрагмент виноситься в новий метод, назва якого пояснює ціль коду. Наприклад, логіку розрахунку необхідної кількості води винесено в метод `calculateWaterAmount()`.
- Rename Variable (Перейменування змінної): Використовується для заміни незрозумілих назв змінних на такі, що розкривають їх суть. Наприклад, заміна змінної `t` на `temperature` або `w` на `waterVolume`.

3. ПРИКЛАДИ

3.1. Код "До" рефакторингу
Модуль обробки даних, де логіка змішана і змінні названі незрозуміло.

public void process() {
    double t = getT();
    // якщо температура більше 30 і вологість менше 20
    if (t > 30 && getH() < 20) {
        System.out.println("Alert!");
        double w = (30 - t) * 5; // якась формула
        pump(w);
    }
}

3.2. Код "Після" рефакторингу
Застосовано Extract Method та Rename Variable.

public void processSensorData() {
    double currentTemperature = getTemperature();
    double currentHumidity = getHumidity();

    if (isCriticalCondition(currentTemperature, currentHumidity)) {
        sendAlert("Critical environmental conditions detected!");
        double waterAmount = calculateWaterAmount(currentTemperature);
        activatePump(waterAmount);
    }
}

private boolean isCriticalCondition(double temp, double humidity) {
    return temp > 30 && humidity < 20;
}

private double calculateWaterAmount(double temp) {
    return (temp - 30) * 5; // Формула розрахунку води на основі перевищення температури
}

ВИСНОВКИ
У ході виконання роботи було опановано основи рефакторингу та принципи роботи з системою контролю версій Git. Використання Git Flow дозволяє безпечно розробляти нові функції у ізольованих гілках. Застосування технік рефакторингу, таких як виділення методів та перейменування змінних, робить код проєкту "Smart Plant Care System" зрозумілішим, чистішим та легшим для подальшого розширення фенкціоналу.
