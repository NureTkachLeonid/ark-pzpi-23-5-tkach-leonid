1 ТЕМА ТА ЗАВДАННЯ

Тема: Розробка IoT клієнта.
Завдання: Розробити програмний клієнт, що емулює роботу IoT-пристрою (розумний горщик), зчитує дані з сенсорів та відправляє їх на сервер за допомогою REST API.

________________

2 ОБҐРУНТУВАННЯ ВИБОРУ ЗАСОБІВ

Для реалізації клієнтської частини було обрано мову програмування Python замість запланованої раніше C++. Основні причини такого рішення:

1.  **Висока швидкість розробки прототипу**: Python дозволяє написати функціональний скрипт емуляції сенсорів за лічені хвилини завдяки лаконічному синтаксису.
2.  **Простота роботи з мережею**: Бібліотека `requests` є стандартом де-факто для роботи з HTTP-запитами, що значно спрощує взаємодію з REST API сервером порівняно з C++ бібліотеками (як-от cURL або Boost.Asio).
3.  **Зручність роботи з JSON**: Вбудована підтримка структур даних (dict, list) та легка серіалізація в JSON роблять Python ідеальним для IoT-прототипів.
4.  **Контейнеризація**: Python-скрипти легко упаковуються в Docker-контейнери, що відповідає сучасним практикам розгортання мікросервісів та IoT-рішень.

________________

3 ОПИС РЕАЛІЗАЦІЇ

Реалізація IoT-клієнта складається з декількох компонентів, розміщених у директорії `iot-client`:

-   **`config.py`**: Відповідає за налаштування параметрів пристрою. Зчитує змінні оточення (SERVER_URL, DEVICE_ID, SEND_INTERVAL), що дозволяє гнучко налаштовувати клієнт без зміни коду (наприклад, через docker-compose).
-   **`main.py`**: Містить основну логіку роботи пристрою.
    -   Функція `generate_sensor_data()` генерує випадкові, але реалістичні показники (вологість 30-80%, температура 15-30°C).
    -   Основний цикл програми виконує генерацію даних, відправку POST-запиту на сервер та очікування протягом заданого інтервалу.
    -   Реалізовано обробку помилок з’єднання, щоб пристрій не припиняв роботу у разі недоступності сервера.
-   **`Dockerfile`**: Описує середовище для запуску клієнта на базі `python:3.9-slim`, що забезпечує ізоляцію та легке розгортання.

4 ВИСНОВКИ

В ході виконання лабораторної роботи було успішно розроблено та протестовано IoT-клієнт на мові Python. Було набуто практичних навичок роботи з HTTP-запитами, емуляції роботи сенсорів та контейнеризації Python-додатків. Реалізоване рішення повністю інтегрується з раніше розробленим бекендом, забезпечуючи наповнення бази даних актуальною телеметрією.
